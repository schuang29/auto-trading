## Proposed strategy adjustments (for human review — not applied)

The week's lessons are about *reliability and observability*, not strategy weights. No `strategy/`, `guardrails/`, or `routines/` file was modified.

1. **Add a dead-man's-switch / heartbeat alert (HIGHEST PRIORITY).** Observation: the bot went dark for three consecutive trading days (05-13→05-15) with no notification; the failure surfaced only on a manual Sun 05-17 review. Propose an external watchdog: if no `portfolio_daily.csv` row appears for a given trading date by ~17:30 ET (or no EOD log section is written), send a notification (the `skills/notifications` channel) so a missed day is known within hours, not days. A bot whose failure mode is silent invisibility is more dangerous than one that occasionally errors loudly.

2. **Fix the `@-` heredoc write bug — escalate from W19 proposal #2.** Observation: the artifact recurred on 05-12 pre-market and this time destroyed the regime signal on a CPI day. This is no longer cosmetic. Recommend a concrete fix in the wrapper / log-append path (`scripts/run_pre_market.ps1` or wherever the brief is appended) plus a post-write assertion that the pre-market section contains a parseable `**Regime:**` line; if not, retry or alert.

3. **Root-cause and fix the `wincredman` git-push credential failure.** Observation: all three 05-12 routines failed to push; commits are local-only. The audit trail's durability depends on the remote. Recommend switching the push to a credential method that does not depend on the interactive Windows Credential Manager (e.g., a scoped PAT in the gitignored `.env`, or SSH deploy key), and having the routine *flag in the daily log and notify* when a push fails (it currently only writes a local "Note:" that itself never gets pushed).

4. **Add idempotent date-guarding to the timeseries writers.** Observation: a `2026-05-17` (Sunday) row exists in all three CSVs. Propose the timeseries writers refuse to write a row for a date that is not a NYSE trading day (or stamp catch-up runs with the trading date they are backfilling, not `today()`), to prevent non-trading-day pollution and keep weekly-return math well-defined.

5. **(Carry forward, no new action) W19 proposals #1, #3, #4, #5** remain open and unaddressed. #3 (EOD write-completion check) is now clearly worth doing given this week's broader output failures. #4 (steady-state idle): Option A remains the implicit, defensible choice — the zero-trade weeks are by design, not a bug. #5 (VTI-at-cap): VTI sat at 30.0–30.2% all instrumented days; still no sell-side rule, still worth pre-defining a trim threshold before it becomes acute.
